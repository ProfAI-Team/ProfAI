## Task Yönetim Akışı

1. **Faz başlar:** `docs/roadmap/phase-N-*.md` içindeki acceptance criteria temel alınır → `phase-N-breakdown.md` yazılır.
2. **Task'lar öncelik + tahmini süre + sahip ile listelenir.**
3. **Her task kapsamında:** PR hedefi, dosya değişimleri, test kriteri net.
4. **Faz bitiminde:** retro yazılır; `docs/roadmap/phase-N-*.md` güncellenir ("gerçekleşen" bölümü).
